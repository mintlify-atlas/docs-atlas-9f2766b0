This page covers everything you need to install SWCE from scratch: what the server must have, how to configure the environment, how to build the database, and how to get the frontend assets ready to serve.

## System requirements

SWCE requires the following on the host server:

| Requirement | Minimum version | Notes |
|---|---|---|
| PHP | 8.2 | Extensions: `pdo_mysql`, `mbstring`, `openssl`, `fileinfo`, `xml`, `zip` |
| MySQL | 8 | Used for app data, sessions, cache, and queues |
| Composer | 2 | PHP dependency manager |
| Node.js | 18 | Required to build frontend assets |
| npm | 9 | Bundled with Node.js 18+ |

## Environment configuration

Copy `.env.example` to `.env` and set values for your environment:

```bash
cp .env.example .env
```

<AccordionGroup>
  <Accordion title="Application settings">

    |---|---|---|
    | `APP_NAME` | `SWCE` | Application name shown in the UI and emails |
    | `APP_ENV` | `local` | Set to `production` for production deployments |
    | `APP_KEY` | _(empty)_ | Generated by `php artisan key:generate` |
    | `APP_DEBUG` | `true` | Set to `false` in production |
    | `APP_URL` | `http://swce.test` | The full URL of your installation |
    | `APP_LOCALE` | `es` | Default application locale |
    | `APP_FALLBACK_LOCALE` | `en` | Fallback locale when a translation is missing |

    | `DB_PORT` | `3306` | Database server port |

## Database setup

Create the database before running migrations:

```sql
CREATE DATABASE swce CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### Run migrations

```bash
php artisan migrate
```

Migrations create the following tables, in order:

| Table group | Tables created |
|---|---|
| Core Laravel | `users`, `cache`, `jobs`, `sessions` |
| Authentication | Two-factor columns on `users` |
| Permissions | `roles`, `permissions`, `model_has_roles`, `model_has_permissions`, `role_has_permissions` |
| Reference data | `countries`, `meses`, `dias`, `licenciaturas`, `cuatrimestres`, `generaciones` |
| Academic records | `alumnos`, `datos_contactos`, `datos_escolares`, `inscripciones` |
| Teaching | `profesores`, `materias`, `asignacion_materias`, `horarios` |
| Grades and cycles | `calificaciones`, `ciclo_escolares` |
| Documents | `documentacion` |
| Dashboards | `dashboard_estudiantes`, `dashboard_profesors` |

### Seed the database

```bash
php artisan db:seed
```

The seeder runs the following in sequence:

| Seeder | What it creates |
|---|---|
| `RoleSeeder` | Three roles (`Admin`, `Profesor`, `Estudiante`) and all scoped permissions for each |
| `DiaSeeder` | Days of the week reference records |
| `CountriesTableSeeder` | Country records (via `altwaireb/laravel-countries`) |
| `LicenciaturaSeeder` | Degree program catalog entries |
| `MesSeeder` | Month reference records |
| `CuatrimestreSeeder` | Academic period (cuatrimestre) catalog entries |
| `GeneracionSeeder` | Generation (cohort year) catalog entries |
| `UserSeeder` | Sample users — runs only in `local` and `testing` environments |
| Default admin | Creates or updates a user with the `Admin` role using `DEFAULT_ADMIN_EMAIL` and `DEFAULT_ADMIN_PASSWORD` from `.env` (defaults: `admin@swce.com` / `Swce#2026`) |

<Warning>
  The `RoleSeeder` uses `Permission::create()` and will fail if run a second time because permission names must be unique. If you need to re-seed a database, truncate the Spatie permission tables first or use `php artisan migrate:fresh --seed`.
</Warning>

## Building frontend assets

SWCE uses Vite for asset bundling. Build the production assets before serving the application:

```bash
npm install
npm run build
```

Built assets are written to the `public/build/` directory and served directly by your web server.

## Cloudflare Turnstile CAPTCHA

SWCE includes `laragear/turnstile` for Cloudflare Turnstile CAPTCHA support. If you enable Turnstile on your login or registration forms, you must add your site credentials to `.env`:

<Note>
  Turnstile requires a Cloudflare account. Create a site key and secret key in the [Cloudflare dashboard](https://dash.cloudflare.com/) under **Turnstile**, then add them to your environment before deploying forms that use the `<x-turnstile />` component.
</Note>

```ini
TURNSTILE_SITE_KEY=your_site_key_here
TURNSTILE_SECRET_KEY=your_secret_key_here
```

Without valid keys, any form that renders the Turnstile widget will fail validation.
