---
title: "Get started with SWCE"
description: "Clone SWCE, install PHP and Node dependencies, configure the database environment, run migrations and seeds, and log in as admin in under ten minutes."
---

This guide walks you through running SWCE on your local machine from a fresh clone to a working login. The entire process takes about five minutes assuming prerequisites are already installed.

## Prerequisites

Make sure you have the following installed before you begin:

| Requirement | Minimum version |
|---|---|
| PHP | 8.2 |
| Composer | 2 |
| Node.js | 18 |
| MySQL | 8 |

## Setup steps

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/xxRevenantx/SWCE.git
    cd SWCE
    ```
  </Step>

  <Step title="Install PHP dependencies">
    ```bash
    composer install
    ```
  </Step>

  <Step title="Configure your environment">
    Copy the example environment file and open it in your editor:

    ```bash
    cp .env.example .env
    ```

    Set your database credentials:

    ```ini
    DB_DATABASE=swce
    DB_USERNAME=your_db_user
    DB_PASSWORD=your_db_password
    ```

    The default `APP_URL` in `.env.example` is `http://swce.test`. Change it to `http://localhost:8000` if you are using `php artisan serve`.
  </Step>

  <Step title="Generate the application key">
    ```bash
    php artisan key:generate
    ```
  </Step>

  <Step title="Run database migrations">
    ```bash
    php artisan migrate
    ```

    This creates all tables — users, permissions, students, professors, subjects, schedules, grades, enrollments, and more.
  </Step>

  <Step title="Seed the database">
    ```bash
    php artisan db:seed
    ```

    The seeder loads roles and permissions, countries, degree programs (licenciaturas), days, months, cohorts (cuatrimestres), generations, and an initial admin user. See [Installation](/installation) for details on what each seeder creates.
  </Step>

  <Step title="Start the development servers">
    ```bash
    composer dev
    ```

    <Note>
      The `composer dev` script uses `concurrently` to run three processes at the same time: `php artisan serve` (the web server), `php artisan queue:listen --tries=1` (the job queue), and `npm run dev` (the Vite asset server). All three must be running for the application to work correctly in development.
    </Note>
  </Step>

  <Step title="Log in">
    Open [http://localhost:8000](http://localhost:8000) in your browser.

    The default admin credentials created by the seeder are:

    | Field | Value |
    |---|---|
    | Email | `admin@swce.com` |
    | Password | `Swce#2026` |

    You can override these before seeding by setting `DEFAULT_ADMIN_EMAIL` and `DEFAULT_ADMIN_PASSWORD` in your `.env` file.
  </Step>
</Steps>
